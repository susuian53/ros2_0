## YamlParam.msg 字段详解

### 1. 常量定义（参数类型）

常量 值 含义 DOUBLE 1 double 类型参数 S64 2 int64 类型参数 VEC\_X\_DOUBLE 3 vector <double> 类型参数（数组）</double> MAT\_X\_DOUBLE 4 matrix <double> 类型参数（矩阵）</double>

### 2. 字段详解 string name —— 参数名称

- 含义 ：要修改的控制参数的名称
- 可赋值 ：字符串，对应控制算法中的参数名
- 常用参数名 ：
  - use\_rc —— 控制模式切换（0=gamepad模式，1=RC模式）
  - control\_mode —— 运控模式（如 11=行走模式，12=恢复站立）
  - des\_roll\_pitch\_height —— 期望姿态（roll, pitch, height）
  - vel\_des —— 期望速度
  - rpy\_des —— 期望姿态角
  - 其他控制参数... uint64 kind —— 参数类型
- 含义 ：指示该参数的数据类型
- 可赋值 ：
  - 1 —— double 类型（使用 double\_value 字段）
  - 2 —— int64 类型（使用 s64\_value 字段）
  - 3 —— vector <double> 类型（使用 vecxd\_value 字段，数组长度12）</double>
  - 4 —— matrix <double> 类型（矩阵）</double>
- 影响 ：必须与 name 对应的参数实际类型匹配，否则参数修改无效 int64 s64\_value —— int64 类型值
- 含义 ：当 kind = 2 时，使用该字段传递整数值
- 可赋值 ：任意 int64 范围内的整数
- 常用值 ：
  - control\_mode = 7 —— 缓慢趴下模式
  - control\_mode = 11 —— locomotion 行走模式
  - control\_mode = 12 —— 恢复站立模式
  - use\_rc = 0 —— 切换到 gamepad 控制模式（必须先发送此指令才能控制）
  - use\_rc = 1 —— RC 控制模式
- 影响 ：直接改变控制程序的运行模式或状态 float64 double\_value —— double 类型值
- 含义 ：当 kind = 1 时，使用该字段传递浮点数值
- 可赋值 ：任意 float64 范围内的浮点数
- 常用场景 ：设置单个浮点参数，如步态参数、控制增益等
- 影响 ：修改控制算法的具体数值参数 float64\[12] vecxd\_value —— 向量类型值
- 含义 ：当 kind = 3 时，使用该字段传递长度为12的浮点数组
- 可赋值 ：12个浮点数的数组
- 常用场景 ：
  - des\_roll\_pitch\_height —— 设置期望 roll、pitch、高度（通常只用前3个元素）
  - vel\_des —— 期望速度 \[x, y, yaw]
  - rpy\_des —— 期望姿态角 \[roll, pitch, yaw]
  - foot\_pose —— 足端位置偏移（6个值，对应4条腿）
  - step\_height —— 抬腿高度（2个值）
- 影响 ：同时修改多个相关的控制参数 int64 is\_user —— 参数来源标识 ⭐关键字段
- 含义 ：指定该参数应写入哪个配置文件
- 可赋值 ：
  - 0 —— 使用 cyberdog2-defaults.yaml （默认参数文件）
  - 非0 （如 1 ）—— 使用 cyberdog2-ctrl-user-parameters.yaml （用户自定义参数文件）
- 对仿真运行的影响 ：
  - is\_user = 0 ：修改默认参数，影响当前运行实例，但不会保存为用户自定义配置
  - is\_user ≠ 0 ：修改用户参数文件，可以用于保存自定义配置，供后续仿真使用
  - 重要 ：在 cyberdogmsg\_sender.cpp 示例中，通常设置为 0 来实时修改当前运行的控制参数

### 3. 使用示例

根据文档中的示例代码：

```
// 修改 control_mode 为 11（行走模式）
auto param = cyberdog_msg::msg::YamlParam();
param.name = "control_mode";
param.kind = 2;        // S64 类型
param.s64_value = 11;  // locomotion 模式
param.is_user = 0;     // 使用默认参数文件
para_pub_->publish(param);

// 修改姿态（roll=0.2, height=0.25）
auto param2 = cyberdog_msg::msg::YamlParam();
param2.name = "des_roll_pitch_height";
param2.kind = 3;       // VEC_X_DOUBLE 类型
param2.vecxd_value = {0.2, 0, 0.25, 0, 0, 0, 
0, 0, 0, 0, 0, 0};
param2.is_user = 0;
para_pub_->publish(param2);
```

### 4. 关键注意事项

1. 必须先发送 use\_rc = 0 ：在发送其他控制指令前，必须先通过 /yaml\_parameter 发送 use\_rc = 0 ，否则控制指令无效
2. 参数类型必须匹配 ： kind 字段必须与参数实际类型匹配，否则参数修改不会生效
3. is\_user 的作用 ：决定参数写入哪个 YAML 文件，影响参数的持久化存储位置
4. 实时生效 ：通过 /yaml\_parameter 发送的参数会立即生效，无需重启仿真
