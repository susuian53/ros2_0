## 1. 环境概述

### 1.1 系统要求

- **操作系统**: Ubuntu 20.04 LTS
- **ROS2**: Galactic
- **Gazebo**: 11.14.0
- **LCM**: 1.5.0
- **Eigen**: 3.3.7
- **Python**: 3.8.10

### 1.2 工程目录结构

```
cyberdog_sim/
├── src/
│   ├── cyberdog_simulator/    # Gazebo 仿真器与可视化工具
│   │   ├── cyberdog_gazebo/   # Gazebo 插件与启动文件
│   │   ├── cyberdog_example/  # 官方示例程序
│   │   ├── cyberdog_visual/   # RViz2 可视化
│   │   └── cyberdog_robot/    # 机器人模型描述
│   ├── cyberdog_locomotion/   # 运动控制算法
│   └── mycode/                # 用户自定义功能包（本文重点）
├── build/                     # 编译输出目录
├── install/                   # 安装目录
├── log/                       # 编译日志
├── launchsim_safe.py          # 一键启动脚本
└── 文档/                       # 文档目录
```

### 1.3 通信架构

```
用户程序 (mycode)
    │
    ├──► ROS2 Topic (/yaml_parameter) ──► Gazebo ──► 控制程序
    │
    └──► LCM (gamepad_lcmt) ────────────► Gazebo ──► 控制程序
```

## 2. 启动仿真

### 2.1 一键启动（推荐）

```bash
cd /home/cyberdog_sim
python3 launchsim_safe.py [选项]
```

#### 启动选项

| 命令 | 说明 |
|------|------|
| `python3 launchsim_safe.py` | 基础仿真（IMU + 关节状态） |
| `python3 launchsim_safe.py --lidar` | 启用激光雷达 |
| `python3 launchsim_safe.py --camera` | 启用 RGB 摄像头 |
| `python3 launchsim_safe.py --lidar --camera` | 启用所有传感器 |

#### 启动流程

1. **清理旧进程** - 自动杀死残留的 Gazebo、RViz2 等进程
2. **启动 Gazebo** - 加载仿真环境和机器人模型
3. **启动可视化** - 打开 RViz2
4. **启动控制程序** - 运行运动控制算法

#### 查看日志

```bash
# 实时查看 Gazebo 日志
tail -f /home/cyberdog_sim/logs/gazebo.log

# 实时查看控制程序日志
tail -f /home/cyberdog_sim/logs/control.log
```

### 2.2 分别启动（调试用）

如果需要单独调试某个模块，可以分别启动：

**终端 1 - Gazebo 仿真:**
```bash
cd /home/cyberdog_sim
source /opt/ros/galactic/setup.bash
source install/setup.bash
ros2 launch cyberdog_gazebo race_gazebo.launch.py use_lidar:=true use_camera:=true
```

**终端 2 - 控制程序:**
```bash
cd /home/cyberdog_sim
source /opt/ros/galactic/setup.bash
source install/setup.bash
ros2 launch cyberdog_gazebo cyberdog_control_launch.py
```

**终端 3 - 可视化:**
```bash
cd /home/cyberdog_sim
source /opt/ros/galactic/setup.bash
source install/setup.bash
ros2 launch cyberdog_visual cyberdog_visual.launch.py
```

### 2.3 停止仿真

- 在启动脚本的终端按 `Ctrl+C`
- 或手动执行: `pkill -9 gzclient gzserver rviz2`

### 3.3 创建新的可执行文件

#### 步骤 1: 编写源代码

在 `src/` 目录下创建新的 `.cpp` 文件，例如 `my_controller.cpp`:

```cpp
#include <iostream>
#include <unistd.h>
#include <rclcpp/rclcpp.hpp>
#include <cyberdog_msg/msg/yaml_param.hpp>
#include <lcm/lcm-cpp.hpp>
#include "gamepad_lcmt.hpp"

class MyController : public rclcpp::Node {
public:
    MyController() : Node("my_controller") {
        // 创建参数发布者
        para_pub_ = this->create_publisher<cyberdog_msg::msg::YamlParam>("yaml_parameter", 10);
    }

    void sendParam(const std::string& name, uint64_t kind, int64_t value) {
        auto msg = cyberdog_msg::msg::YamlParam();
        msg.name = name;
        msg.kind = kind;
        msg.s64_value = value;
        msg.is_user = 0;
        para_pub_->publish(msg);
    }

private:
    rclcpp::Publisher<cyberdog_msg::msg::YamlParam>::SharedPtr para_pub_;
};

int main(int argc, char** argv) {
    rclcpp::init(argc, argv);
    auto node = std::make_shared<MyController>();
    
    // 初始化 LCM
    lcm::LCM lcm;
    if (!lcm.good()) {
        std::cerr << "LCM initialization failed!" << std::endl;
        return 1;
    }

    // 1. 切换到 gamepad 控制模式（必须）
    std::cout << "Switching to gamepad mode..." << std::endl;
    node->sendParam("use_rc", 2, 0);  // kind=2 (S64)
    sleep(1);

    // 2. 恢复站立
    std::cout << "Recovery stand..." << std::endl;
    node->sendParam("control_mode", 2, 12);
    sleep(3);

    // 3. 进入运动模式
    std::cout << "Enter locomotion mode..." << std::endl;
    node->sendParam("control_mode", 2, 11);
    sleep(1);

    // 4. 发送运动指令（通过 LCM）
    gamepad_lcmt gamepad_msg;
    memset(&gamepad_msg, 0, sizeof(gamepad_msg));
    
    // 前进 0.3 m/s
    gamepad_msg.leftStickAnalog[1] = 0.3;
    for (int i = 0; i < 100; i++) {
        lcm.publish("gamepad_lcmt", &gamepad_msg);
        usleep(10000);  // 10ms
    }

    // 停止
    gamepad_msg.leftStickAnalog[1] = 0;
    lcm.publish("gamepad_lcmt", &gamepad_msg);
    
    // 恢复站立
    sleep(1);
    node->sendParam("control_mode", 2, 12);

    rclcpp::shutdown();
    return 0;
}
```

#### 步骤 2: 修改 CMakeLists.txt

  ${CMAKE_CURRENT_SOURCE_DIR}/src

### 3.4 CMakeLists.txt 配置详解

#### 常用命令

| 命令 | 说明 | 示例 |
|------|------|------|
| `find_package` | 查找依赖包 | `find_package(rclcpp REQUIRED)` |
| `add_executable` | 添加可执行文件 | `add_executable(my_node src/my_node.cpp)` |
| `target_include_directories` | 添加头文件路径 | `target_include_directories(my_node PRIVATE include/)` |
| `target_link_libraries` | 链接库 | `target_link_libraries(my_node lcm)` |
| `ament_target_dependencies` | ROS2 依赖 | `ament_target_dependencies(my_node rclcpp)` |
| `install` | 安装目标 | `install(TARGETS my_node DESTINATION lib/${PROJECT_NAME})` |

#### 完整示例模板

```cmake
cmake_minimum_required(VERSION 3.8)
project(my_package)

# 编译选项
set(CMAKE_CXX_STANDARD 14)
if(CMAKE_COMPILER_IS_GNUCXX OR CMAKE_CXX_COMPILER_ID MATCHES "Clang")
  add_compile_options(-Wall -Wextra -Wpedantic)

# 查找依赖

find_package(std_msgs REQUIRED)

# 包含目录
include_directories(
  ${CMAKE_CURRENT_SOURCE_DIR}/include
  ${CMAKE_CURRENT_SOURCE_DIR}/src
)

# 创建可执行文件
add_executable(${PROJECT_NAME}_node 
  src/main.cpp
  src/utils.cpp
)

# 链接库
target_link_libraries(${PROJECT_NAME}_node 
  lcm
  pthread
)

# ROS2 依赖
ament_target_dependencies(${PROJECT_NAME}_node 
  rclcpp
  std_msgs
  cyberdog_msg
)

# 安装
install(TARGETS
  ${PROJECT_NAME}_node
  DESTINATION lib/${PROJECT_NAME}
)

# 安装 launch 文件
install(DIRECTORY
  launch
  DESTINATION share/${PROJECT_NAME}
)

ament_package()
```

## 5. 传感器使用

### 5.1 可用传感器

| 传感器 | Topic | 消息类型 | 启动参数 |
|--------|-------|----------|----------|
| IMU | `/imu` | `sensor_msgs/Imu` | 默认启用 |
| 关节状态 | `/joint_states` | `sensor_msgs/JointState` | 默认启用 |
| 激光雷达 | `/scan` | `sensor_msgs/LaserScan` | `--lidar` |
| RGB 摄像头 | `/camera/rgb/image_raw` | `sensor_msgs/Image` | `--camera` |

### 5.2 在代码中订阅传感器数据

```cpp
#include <rclcpp/rclcpp.hpp>
#include <sensor_msgs/msg/imu.hpp>
#include <sensor_msgs/msg/laser_scan.hpp>
#include <sensor_msgs/msg/image.hpp>

class SensorListener : public rclcpp::Node {
public:
    SensorListener() : Node("sensor_listener") {
        // 订阅 IMU
        imu_sub_ = this->create_subscription<sensor_msgs::msg::Imu>(
            "imu", 10, 
            std::bind(&SensorListener::imuCallback, this, std::placeholders::_1));
        
        // 订阅激光雷达
        scan_sub_ = this->create_subscription<sensor_msgs::msg::LaserScan>(
            "scan", 10,
            std::bind(&SensorListener::scanCallback, this, std::placeholders::_1));
        
        // 订阅摄像头
        image_sub_ = this->create_subscription<sensor_msgs::msg::Image>(
            "/camera/rgb/image_raw", 10,
            std::bind(&SensorListener::imageCallback, this, std::placeholders::_1));
    }

private:
    void imuCallback(const sensor_msgs::msg::Imu::SharedPtr msg) {
        RCLCPP_INFO(this->get_logger(), 
            "IMU: linear_acc=[%.2f, %.2f, %.2f]",
            msg->linear_acceleration.x,
            msg->linear_acceleration.y,
            msg->linear_acceleration.z);
    }
    
    void scanCallback(const sensor_msgs::msg::LaserScan::SharedPtr msg) {
        RCLCPP_INFO(this->get_logger(), 
            "LaserScan: ranges size=%zu", msg->ranges.size());
    }
    
    void imageCallback(const sensor_msgs::msg::Image::SharedPtr msg) {
        RCLCPP_INFO(this->get_logger(), 
            "Image: %dx%d", msg->width, msg->height);
    }
    
    rclcpp::Subscription<sensor_msgs::msg::Imu>::SharedPtr imu_sub_;
    rclcpp::Subscription<sensor_msgs::msg::LaserScan>::SharedPtr scan_sub_;
    rclcpp::Subscription<sensor_msgs::msg::Image>::SharedPtr image_sub_;
};
```

### 5.3 控制指令参考

#### 参数类型 (YamlParam)

| kind 值 | 类型 | 使用字段 |
|---------|------|----------|
| 1 | double | `double_value` |
| 2 | int64 | `s64_value` |
| 3 | vector<double> | `vecxd_value[12]` |
| 4 | matrix<double> | - |

#### 常用控制参数

| 参数名 | 类型 | 说明 | 常用值 |
|--------|------|------|--------|
| `use_rc` | int64 | 控制模式 | 0=gamepad, 1=RC |
| `control_mode` | int64 | 控制状态 | 11=运动模式, 12=恢复站立 |
| `des_roll_pitch_height` | vector | 期望姿态 | [roll, pitch, height, ...] |
| `vel_des` | vector | 期望速度 | [x, y, yaw, ...] |

#### LCM gamepad_lcmt 结构

```cpp
struct gamepad_lcmt {
    int32_t leftBumper, rightBumper;
    int32_t leftTriggerButton, rightTriggerButton;
    int32_t back, start;
    int32_t a, b, x, y;              // 按钮
    int32_t leftStickButton, rightStickButton;
    float leftTriggerAnalog, rightTriggerAnalog;
    float leftStickAnalog[2];        // [0]=左右, [1]=前后
    float rightStickAnalog[2];       // [0]=旋转, [1]=俯仰
};
```

#### 摇杆控制映射

| 摇杆 | 索引 | 方向 | 正值 | 负值 |
|------|------|------|------|------|
| leftStickAnalog[0] | 0 | 左右 | 左移 | 右移 |
| leftStickAnalog[1] | 1 | 前后 | 前进 | 后退 |
| rightStickAnalog[0] | 0 | 旋转 | 左转 | 右转 |
| rightStickAnalog[1] | 1 | 俯仰 | 抬头 | 低头 |

## 6. 常见问题

### Q1: 控制指令无响应

**原因**: 未切换到 gamepad 模式

**解决**:
```cpp
// 必须先发送此指令
node->sendParam("use_rc", 2, 0);
```

### Q2: Gazebo 启动失败

**原因**: 残留进程占用端口

**解决**:
```bash
pkill -9 gzclient gzserver
python3 launchsim_safe.py
```

### Q3: 编译报错找不到头文件

**原因**: 未正确配置 include 路径

**解决**:
```cmake
target_include_directories(my_node PRIVATE
  ${CMAKE_CURRENT_SOURCE_DIR}/include
  ${CMAKE_CURRENT_SOURCE_DIR}/src
)
```

### Q4: 运行时报错找不到共享库

**原因**: 未 source 环境

**解决**:
```bash
source /opt/ros/galactic/setup.bash
source install/setup.bash
```

### Q5: LCM 消息发送失败

**原因**: LCM 初始化失败或网络问题

**解决**:
```cpp
lcm::LCM lcm;
if (!lcm.good()) {
    std::cerr << "LCM initialization failed!" << std::endl;
    return 1;
}
```

### Q6: 如何查看所有可用 Topic

```bash
ros2 topic list
ros2 topic list -t  # 显示消息类型
```

### Q7: 如何录制和回放数据

```bash
# 录制
ros2 bag record /imu /scan /joint_states

# 回放
ros2 bag play <bag_file>
```
