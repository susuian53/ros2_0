# Cyberdog SIM 仿真开发指南

> 本指南基于 [Cyberdog Gazebo 仿真平台文档](https://miroboticslab.github.io/blogs/#/cn/cyberdog_gazebo_cn) 及 [Cyberdog 运动控制二次开发接口文档](https://miroboticslab.github.io/blogs/#/cn/cyberdog_loco_cn) 编写，适用于 `cyberdog_sim` 仓库的 Gazebo + ROS2 仿真环境。

---

## 1. 概述

`cyberdog_sim` 整合 `cyberdog_locomotion` 与 `cyberdog_simulator` 两个仓库，实现 Gazebo 与 ROS2 环境下的四足机器人仿真。平台提供基于 Rviz2 的可视化工具，可将机器人状态的 LCM 数据转发为 ROS2 topics，便于开发者进行算法验证与二次开发。

### 1.1 推荐环境

- **操作系统**：Ubuntu 20.04 LTS
- **ROS 版本**：ROS2 Galactic
- **仿真器**：Gazebo 11
- **Python**：3.8+

---

## 2. 依赖环境

本环境已预装全部所需依赖，无需手动安装：

| 依赖项 | 版本 |
|--------|------|
| ROS2 | Galactic |
| Gazebo | 11.14.0 |
| LCM | 1.5.0 |

## 3. 下载与编译

### 3.1 下载代码

```bash
git clone https://github.com/MiRoboticsLab/cyberdog_sim.git
cd cyberdog_sim
vcs import < cyberdog_sim.repos
```

下载完成后，目录结构如下：

```
cyberdog_sim/
├── src/
│   ├── cyberdog_simulator/   # Gazebo 仿真器与可视化工具
│   └── cyberdog_locomotion/  # 运动控制算法
├── cyberdog_sim.repos
└── ...
```

### 3.2 编译

编译前需将 `src/cyberdog_locomotion/CMakeLists.txt` 中的 `BUILD_ROS` 置为 `ON`：

```cmake
set(BUILD_ROS ON)
```

然后在 `cyberdog_sim` 文件夹下执行：

```bash
source /opt/ros/galactic/setup.bash
colcon build --merge-install --symlink-install --packages-up-to cyberdog_locomotion cyberdog_simulator
```

## 4. 运行仿真

### 4.1 一键启动（推荐）

在 `cyberdog_sim` 文件夹下运行：

```bash
python3 launchsim_safe.py
```

该脚本会先自动清理所有仿真相关进程（防止端口或资源冲突），然后同时启动 Gazebo 仿真、Cyberdog 控制程序与 Rviz2 可视化界面。

> 若需使用原始启动脚本（不会自动清理进程），可运行：
> ```bash
> python3 src/cyberdog_simulator/cyberdog_gazebo/script/launchsim.py
> ```

### 4.2 启动脚本说明

`launchsim_safe.py` 是对原始 `launchsim.py` 的增强版本，主要改进：

- **自动清理进程**：启动前会调用 `pkill` 结束所有仿真相关进程（`gzclient`、`gzserver`、`rviz2`、`ros2 launch` 等），避免端口占用或共享内存冲突。
- **保留原有逻辑**：清理完成后，按原顺序启动 Gazebo、Rviz2 和控制程序。

### 4.3 分别启动各程序

若需单独调试某个模块，可分别启动：

#### 4.2.1 启动 Gazebo 仿真

```bash
source /opt/ros/galactic/setup.bash
source install/setup.bash
ros2 launch cyberdog_gazebo gazebo.launch.py
```

启用激光雷达：

```bash
ros2 launch cyberdog_gazebo gazebo.launch.py use_lidar:=true
```

#### 4.2.2 启动控制程序

```bash
source /opt/ros/galactic/setup.bash
source install/setup.bash
ros2 launch cyberdog_gazebo cyberdog_control_launch.py
```

#### 4.2.3 启动可视化界面

```bash
source /opt/ros/galactic/setup.bash
source install/setup.bash
ros2 launch cyberdog_visual cyberdog_visual.launch.py
```

## 5. 通信接口

仿真环境中的通信架构由三部分组成：

1. **Cyberdog 控制程序** (`cyberdog_control`)：运行运动控制算法
2. **Gazebo 仿真程序** (`legged_plugins`)：物理仿真与传感器模拟
3. **ROS2 可视化界面** (`cyberdog_visual`)：数据可视化与 Topic 转发

### 5.1 控制程序 ↔ Gazebo（共享内存）

控制程序与 Gazebo 之间通过 **共享内存 (Shared Memory)** 进行通信，Gazebo 创建 host 共享内存，控制程序 attach 到该内存上进行数据交换。通信内容包含 `robotToSim` / `simToRobot` 数据结构。

### 5.2 控制程序 ↔ 外部（LCM 通信）

控制程序通过 **LCM (Lightweight Communications and Marshalling)** 与外部程序通信，支持高层接口与底层接口两种模式。

#### 5.2.1 高层接口 —— 基本控制指令

通过 LCM 通道 `robot_control_cmd` 发送控制指令：

- **URL**：`udpm://239.255.76.67:7671?ttl=255`
- **通道名**：`robot_control_cmd`
- **频率**：2~500 Hz（超时 500ms 触发趴下保护）

**数据结构** (`robot_control_cmd_lcmt`)：

| 字段 | 类型 | 说明 |
|------|------|------|
| `mode` | `int8_t` | 运控状态机模式 |
| `gait_id` | `int8_t` | 子动作 ID |
| `contact` | `int8_t` | 四足触地状态（位控模式用） |
| `life_count` | `int8_t` | 递增心跳信号，命令在数值变化时生效 |
| `vel_des[3]` | `float` | x, y 方向速度 (m/s)，yaw 角速度 (rad/s) |
| `rpy_des[3]` | `float` | roll, pitch, yaw 期望姿态 (rad) |
| `pos_des[3]` | `float` | 期望质心位置偏移 (m) |
| `acc_des[6]` | `float` | 跳跃动作加速度 (m/s²) |
| `ctrl_point[3]` | `float` | 姿态控制中心点坐标 |
| `foot_pose[6]` | `float` | 足端坐标偏移 |
| `step_height[2]` | `float` | 行走时抬腿高度 (0~0.08m) |
| `value` | `int32_t` | 模式相关参数（如是否使用 MPC 轨迹、步态类型） |
| `duration` | `int32_t` | 动作持续时间 (ms)，0 表示持续执行 |

**常用模式与动作**（`mode` / `gait_id`）：

| mode | gait_id | 动作说明 |
|------|---------|----------|
| 7 | 0 | 缓慢趴下 (PureDamper) |
| 11 | 3 | 中速行走 (TROT_MEDIUM) |
| 11 | 10 | 快速行走 (TROT_FAST) |
| 11 | 26 | 自变频行走 |
| 11 | 27 | 慢速行走 (TROT_SLOW) |
| 12 | 0 | 恢复站立 (Recovery Stand) |
| 21 | 0 | 位控姿态模式 (POSE_CTRL) |
| 62 | 2 | 握手 |
| 64 | 0 | 两腿站立 |

#### 5.2.2 高层接口 —— 状态反馈

通过 LCM 通道 `robot_control_response` 接收机器人状态：

- **URL**：`udpm://239.255.76.67:7670?ttl=255`
- **通道名**：`robot_control_response`
- **频率**：50 Hz

**数据结构** (`robot_control_response_lcmt`)：

| 字段 | 类型 | 说明 |
|------|------|------|
| `mode` | `int8_t` | 当前运控模式 |
| `gait_id` | `int8_t` | 当前子动作 |
| `contact` | `int8_t` | 估计的四腿触地状态 |
| `order_process_bar` | `int8_t` | 动作执行进度 (1 = 1%) |
| `switch_status` | `int8_t` | 切换状态：0=完成, 1=转换中, 2=急停, 3=高阻尼, 4=被提起, 5=禁止切换, 6=过热, 7=低电量 |
| `ori_error` | `int8_t` | 倾角超限错误 |
| `footpos_error` | `int16_t` | 足端工作空间错误 |
| `motor_error[12]` | `int32_t` | 12 个关节电机错误状态 |

#### 5.2.3 底层接口 —— 电机直接控制

通过 LCM 通道 `motor_ctrl` 发送电机控制指令，直接控制 12 个关节电机：

**控制指令** (`MotorCmd`)：

| 字段 | 类型 | 说明 |
|------|------|------|
| `q_des[12]` | `float` | 期望关节角度 (rad) |
| `qd_des[12]` | `float` | 期望关节角速度 (rad/s) |
| `kp_des[12]` | `float` | 位置控制比例系数 (0~200) |
| `kd_des[12]` | `float` | 速度控制微分系数 (0~10) |
| `tau_des[12]` | `float` | 前馈力矩 (Nm) |

**反馈数据** (`RobotData`)：

| 字段 | 类型 | 说明 |
|------|------|------|
| `q[12]` | `float` | 关节角度 (rad) |
| `qd[12]` | `float` | 关节角速度 (rad/s) |
| `tau[12]` | `float` | 关节力矩 (Nm) |
| `quat[4]` | `float` | 机身姿态四元数 (xyzw) |
| `rpy[3]` | `float` | 机身 roll/pitch/yaw (rad) |
| `acc[3]` | `float` | IMU 加速度 (m/s²) |
| `omega[3]` | `float` | IMU 角速度 (rad/s) |

### 5.3 仿真程序 ↔ ROS2 Topic

Gazebo 仿真程序将传感器数据与机器人状态以标准 ROS2 Topic 形式发布，同时接收外部控制指令。

#### 5.3.1 发布的 Topic（传感器与状态数据）

| Topic 名称 | 消息类型 | 说明 |
|-----------|---------|------|
| `/imu` | `sensor_msgs/Imu` | IMU 数据（加速度、角速度、姿态） |
| `/scan` | `sensor_msgs/LaserScan` | 激光雷达数据（需启用 `use_lidar:=true`） |
| `/joint_states` | `sensor_msgs/JointState` | 12 个关节的位置、速度、力矩 |
| `/tf` | `tf2_msgs/TFMessage` | 机器人各连杆坐标变换 |

#### 5.3.2 接收的 Topic（控制指令）

| Topic 名称 | 消息类型 | 说明 |
|-----------|---------|------|
| `/yaml_parameter` | `YamlParam` | 实时修改控制参数 |
| `/apply_force` | `ApplyForce` | 对机器人施加外部力（仅仿真可用） |

**`/yaml_parameter` 消息结构** (`YamlParam.msg`)：

```
uint64 DOUBLE = 1
uint64 S64 = 2
uint64 VEC_X_DOUBLE = 3

string name          # 参数名
uint64 kind          # 参数类型：1=double, 2=int64, 3=vector<double>
int64 s64_value      # int64 类型值
float64 double_value # double 类型值
float64[12] vecxd_value # vector 类型值
int64 is_user        # 0=cyberdog2-defaults.yaml, 非0=cyberdog2-ctrl-user-parameters.yaml
```

**`/apply_force` 消息结构** (`ApplyForce.msg`)：

```
string link_name     # 受力构件名称
float64 time         # 力的持续时间 (s)
float64[3] force     # 世界坐标系下的力 (N)
float64[3] rel_pos   # 构件坐标系下的作用点位置
```

### 5.4 仿真程序 ↔ LCM（仿真器数据）

仿真程序通过 LCM 发送 `simulator_state` 数据，包含 Gazebo 中的真实物理状态：

- **通道名**：`simulator_lcmt`

**数据结构** (`simulator_lcmt`)：

| 字段 | 类型 | 说明 |
|------|------|------|
| `p[3]` | `double` | 机身在世界坐标系下的位置 |
| `v[3]` | `double` | 机身在世界坐标系下的速度 |
| `vb[3]` | `double` | 机身坐标系下的速度 |
| `rpy[3]` | `double` | 翻滚角、俯仰角、偏航角 |
| `quat[4]` | `double` | 机身朝向四元数 |
| `R[9]` | `double` | 机身朝向旋转矩阵 |
| `omega[3]` | `double` | 世界坐标系下角速度 |
| `omegab[3]` | `double` | 机身坐标系下角速度 |
| `q[12]` | `double` | 关节位置 |
| `qd[12]` | `double` | 关节速度 |
| `qdd[12]` | `double` | 关节加速度 |
| `tau[12]` | `double` | 关节输出转矩 |
| `tauAct[12]` | `double` | 关节实际转矩 |
| `f_foot[12]` | `double` | 足端接触力 (xyz 分量，顺序：FR, FL, RR, RL) |
| `p_foot[12]` | `double` | 足端位置 (xyz 坐标，顺序：FR, FL, RR, RL) |
| `timesteps` | `int64_t` | 时间戳 |
| `time` | `double` | 仿真时间 |

> **关节顺序**：FR-侧摆髋关节, FR-前摆髋关节, FR-膝关节, FL-侧摆髋关节, FL-前摆髋关节, FL-膝关节, RR-侧摆髋关节, RR-前摆髋关节, RR-膝关节, RL-侧摆髋关节, RL-前摆髋关节, RL-膝关节

## 6. 坐标系与关节定义

### 6.1 机身坐标系

- **x 轴**：指向机器人前方
- **y 轴**：指向机器人左侧
- **z 轴**：指向机器人上方
- 遵循**右手法则**

### 6.2 腿与关节命名

四条腿按以下顺序编号：

1. **FR**：Front-Right（右前腿）
2. **FL**：Front-Left（左前腿）
3. **RR**：Rear-Right（右后腿）
4. **RL**：Rear-Left（左后腿）

每条腿包含 3 个关节：

1. **侧摆髋关节**（Abduction/Adduction）
2. **前摆髋关节**（Hip Flexion/Extension）
3. **膝关节**（Knee Flexion/Extension）

### 6.3 关节零位与范围

关节零位定义：侧摆髋关节水平，前摆髋关节和膝关节组成的腿部垂直向下时为电气零位（因连杆机械限制，无法实际达到）。

| 关节名称 | 范围 [rad] | 最大速度 [m/s] | 最大力矩 [Nm] |
|---------|-----------|---------------|--------------|
| FR/FL-侧摆髋关节 | [-0.68, 0.68] | 38.19 | 12 |
| FR/FL-前摆髋关节 | [2.79, -1.33] | 38.19 | 12 |
| FR/FL-膝关节 | [-0.52, -2.53] | 38.19 | 12 |
| RR/RL-侧摆髋关节 | [-0.68, 0.68] | 38.19 | 12 |
| RR/RL-前摆髋关节 | [3.14, -0.98] | 38.19 | 12 |
| RR/RL-膝关节 | [-0.52, -2.53] | 38.19 | 12 |

## 8. 仿真参数调整

### 8.1 Gazebo 物理引擎参数

在 `cyberdog_gazebo/world/simple.world` 中可调整 ODE 求解器参数：

```xml
<physics type="ode">
    <max_step_size>0.001</max_step_size>
    <real_time_factor>1</real_time_factor>
    <real_time_update_rate>1000</real_time_update_rate>
    <gravity>0 0 -9.81</gravity>
    <ode>
        <solver>
            <type>world</type>       <!-- "world" 精确但慢，"quick" 快但精度低 -->
            <iters>50</iters>        <!-- 迭代次数 -->
            <sor>1.3</sor>
        </solver>
        <constraints>
            <cfm>0.0</cfm>
            <erp>0.2</erp>
        </constraints>
    </ode>
</physics>
```

### 8.2 机器人模型参数

机器人 URDF/Xacro 模型文件位于 `cyberdog_robot/xacro/` 目录下，mesh 文件位于 `cyberdog_robot/mesh/` 目录下。

### 8.3 地形设置

Gazebo 地形可通过 heightmap 生成，需准备大小为 `(2^n+1) × (2^n+1)`（如 513×513）的灰度图：

1. 将地形图像放入 `media/materials/textures/` 文件夹
2. 修改 `world/hightmap.world` 中的纹理路径
3. 启动地形仿真：

```bash
ros2 launch cyberdog_gazebo heightmap_gazebo.launch.py
```

## 9. 实机 LCM 数据回放

仿真平台支持通过 Rviz2 播放和可视化实机的运动控制 LCM 数据。

**操作步骤**：

```bash
# 1. 启动 LCM 数据可视化界面
source /opt/ros/galactic/setup.bash
source install/setup.bash
ros2 launch cyberdog_visual cyberdog_lcm_repaly.launch.py

# 2. 在另一个终端播放 LCM log 数据
cd src/cyberdog_locomotion/scripts
./make_types.sh          # 首次使用需运行
./launch_lcm_logplayer.sh
```

运行后选择需要播放的 LCM log 文件，即可在 Rviz2 中复现机器人姿态。

## 10. 常见问题

| 问题 | 解决方案 |
|------|---------|
| Gazebo 进程无法通过 Ctrl+C 彻底杀死 | 使用 `launchsim_safe.py` 一键启动，会自动清理残留进程；或手动执行 `pkill -9 gzclient gzserver` |
| yaml-cpp 冲突 | 确保系统中无其他版本 yaml-cpp，或临时卸载 |
| 编译失败 | 确认 `BUILD_ROS=ON`，并检查 ROS2 环境是否 source |
| 控制指令无响应 | 检查 `life_count` 是否递增，确认 LCM 通信正常 |
| 仿真卡顿 | 尝试降低 Gazebo 画质，或使用 `gui:=false` 关闭 GUI |
